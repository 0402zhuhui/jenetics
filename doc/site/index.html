Abstract

Jenetics is an Genetic Algorithm Library written in Java. It is designed with a clear separation of the several concepts of the algorithm, e.g. Gene, Chromosome, Genotype, Phenotype, Population and FitnessFunction.

Design

This project provides an Genetic Algorithm (GA) implementation. Genetic Algorithms was first proposed and analyzed by John Holland in 1975.

Data structures

The diagram above shows the main data structures of the GA implementation. The Gene is the base of the building block. Genes are aggregated in Chromosomes. One to n Chromosomes are aggregated in Genotypes. A Genptype and a FitnessFunction form the Phenotype

Simple GA Setup

The minimum GA setup needs a GenotypeFactory and an FitnessFunction. The Genotype implements the GenotypeFactory interface and can be used as prototype.

final Factory<Genotype<BitGene>> gtf = Genotype.valueOf(
        BitChromosome.valueOf(10, 0.5);
    );
final FitnessFunction<BitGene, Float64> ff = ...
final GeneticAlgorithm<BitGene, Float64> ga = 
        GeneticAlgorithm.valueOf(gtf, ff);

The genotype-factory gtf in this example will create genotypes which consits of a bit-chromosome of length 10. The one to zero probability of the newly created genotypes are set to 0.5. The fitness-function is parameterized with BitGene and Float64. That means that the function is calculating the fitness value, which is a Float64, from a bit-chromosome. The calculated fitness value must be at least a Comparable. Now we are done! A new GA instance can be created with the genotype-factory and the fitness-function. For starting the evolution just call ga.setup() and ga.evolve(100).

ga.setup(); 
ga.evolve(100); 
System.out.println(ga);

The GeneticAlgorithm.setup() call creates the initial population and calculates its fitness. Then the GA evolves 100 generations and prints the (last) populations statistics.

In a more advanced setup you want to define mutation and/or selection strategies.

ga.setSelectors(new TournamentSelector<BitGene>(5));
ga.setAlterer(new SinglePointCrossover<BitGene>(0.1));
ga.addAlterer(new Mutation<BitGene>(0.01))

The Alterers are concatenated, first the crossover (with crossover probability 0.1) is performed and than the chromosomes are mutated (with a probability 0.01).
