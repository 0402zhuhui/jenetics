Description

Jenetics is an Genetic Algorithm Library written in Java. Clear separation of the several concepts of the algorithm, e.g. Gene, Chromosome, Genotype, Phenotype, Population and FitnessFunction.

Design

This project provides an Genetic Algorithm (GA) implementation. Genetic Algorithms was first proposed and analyzed by John Holland in 1975. The GA consists of the following steps: 

Initialize population;
Measure fitness of populaton;
while (!stopCondition) {
    select offsprings;
    select survivors;
    increase survirvor age;
    alter offsprings;
    add survivors and offsprings to new population;
    measure fitness of new population;
}

Data structures

The diagram above shows the main data structures of the GA implementation. The Gene is the base of the building block. Genes are aggregated in Chromosomes. One to n Chromosomes are aggregated in Genotypes. A Genptype and a FitnessFunction form the Phenotype. are collected into a Population.

Setting up the Genetic Algorithm

The minimum GA setup needs a GenotypeFactory and an FitnessFunction. The Genotype implements the GenotypeFactory interface and can be used as prototype. 

//Defining a Genotype prototype.
final Factory<Genotype<BitGene>> gtf = Genotype.valueOf(
        BitChromosome.valueOf(10, 0.5);
    );
final FitnessFunction<BitGene, Float64> ff = ...

final GeneticAlgorithm<BitGene, Float64> ga = 
	GeneticAlgorithm.valueOf(gtf, ff);

All other needed GA parameters are initialized with default values. Therefore the GA is ready for use now.

ga.setup(); ga.evolve(100); System.out.println(ga);

The GeneticAlgorithm.setup() call creates the initial population and calculates its fitness. Then the GA evolves 100 generations and prints the (last) populations statistic. In a more advanced setup you want to define mutation and/or selection strategies. 

...
final Selector<BitGene> selector = 
	new StochasticUniversalSelector<BitGene>();
ga.setOffspringSelector(selector);
ga.setSurvivorSelector(selector);
final Alterer alterer = new CompositeAlterer<BitGene>(
        new SinglePointCrossover<BitGene>(0.1),
        new Mutation<BitGene>(0.01)
    );
ga.setAlterer(alterer);

You can define selection strategies for survivors and offsprings independently. The Alterers are concatenated, first the crossover (with crossover probability 0.1) is performed and than the chromosomes are mutated (with a probability 0.01).
